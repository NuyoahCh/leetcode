# Leetcode 题解仓库

基于 Go 与 Java 编写的个人刷题与题解记录。仓库按照专题与时间划分，包含详细的思路解析、代码实现以及必要的测试用例，旨在帮助读者理解不同解题思路与实现技巧。

## 内容特色
- **双语言实现**：同一题目分别使用 Go 和 Java 记录解法，便于对比不同语言的实现细节。
- **按专题分类**：`offer`、`hot100` 等目录覆盖高频与经典题目，`daily` 目录追踪每日一题的思路。
- **可复用模板**：常见算法模板、测试样例与注释齐备，便于在其他项目中复用。
- **持续更新**：保持每日或阶段性刷题节奏，逐步补全不同难度和方向的题目。

## 仓库结构
- `problems/`
  - `offer/`：剑指 Offer 系列题解。
  - `hot100/`：LeetCode 热门 100 题的个人解题记录。
  - `daily/`：每日一题的思路整理与代码实现。
  - `test/`：与题目对应的测试或示例输入输出。
- `pic/`：题解插图、思路草图或流程图等辅助材料。

## 快速开始
1. **克隆仓库**：`git clone https://github.com/<your-name>/leetcode.git && cd leetcode`
2. **安装依赖**：
   - Go ≥ 1.19，确保已配置 `GOPATH` 与 `GOROOT`（可选使用 `go env` 查看）。
   - Java ≥ 11，推荐使用 `sdkman` 或 `jenv` 管理版本。
3. **验证环境**：运行 `go version` 与 `java -version` 确认编译器可用。

## 如何使用
1. **查阅题解**：在对应目录下找到题目文件，阅读注释获取题意、思路和复杂度分析。
2. **运行 Go 代码**：进入题目目录后执行 `go test` 或 `go run <file>.go` 以验证实现。
3. **运行 Java 代码**：使用 `javac` 编译后通过 `java <MainClass>` 运行，或在 IDE 中直接执行。
4. **对比多种方案**：若同一题目存在多份解法文件，可对照查看时间/空间复杂度差异。

## 代码风格与目录约定
- **命名**：文件名保持题目编号与名称，例如 `problems/daily/p123_foo.go`、`p123_foo.java` 便于检索。
- **注释**：在文件开头注明题目链接、时间复杂度与空间复杂度；关键逻辑配备行内注释。
- **测试**：Go 题解优先使用 `_test.go` 组织单元测试；Java 题解可附带 `main` 方法或 JUnit 测试类。
- **图片**：若题解配图，请将素材放入 `pic/` 并在题解注释中注明文件名。

## 贡献指南
欢迎通过 Issue 或 Pull Request 贡献题解、优化现有代码或补充测试：
1. Fork 本仓库并创建分支。
2. 针对单个题目提交修改，确保代码可编译并附上必要注释。
3. 在提交前运行相关测试（如 `go test`）以避免回归。
4. 在 PR 中简要说明解题思路、复杂度和测试范围。
5. PR 描述建议包含：所涉题目编号、语言、思路摘要与测试结论，便于快速 review。
